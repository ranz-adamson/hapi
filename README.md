# HAPI: Enterprise Order-to-Cash (O2C) & Studio Management CRM

## Executive Summary
[cite_start]The HAPI system is a proprietary, bespoke Customer Relationship Management (CRM) and studio management application designed to handle complex, end-to-end business operations[cite: 4, 9]. [cite_start]Developed initially in 2013 and modernized through multiple PHP iterations, it serves as the central operational hub for managing the complete client lifecycle—from automated lead ingestion to production workflows and final financial reconciliation[cite: 5].

This repository outlines the architectural blueprint, data orchestration models, and infrastructure design of the HAPI platform. 

## Architectural Highlights

### 1. Advanced Data Orchestration & Integrations
The system is built to automate tedious manual data entry by orchestrating data from multiple disparate sources:
* [cite_start]**Automated Lead Ingestion:** Establishes direct, secure remote database connections to ingest leads from external WordPress sources (Gravity Forms, Visual Form Builder)[cite: 28, 30, 31, 77]. 
* [cite_start]**Complex File Parsing (ProSelect):** Automates order creation by parsing uploaded `.zip` archives containing complex XML data and high-resolution images generated by third-party ProSelect software.
* [cite_start]**Automated Task Generation:** Dynamically generates bespoke production tasks (e.g., "Edit Images", "Order Prints") in the `todolist` table based on the specific SKU/products purchased in the parsed order[cite: 57, 58, 59].

### 2. Metadata-Driven CRUD Interface
[cite_start]Rather than hardcoding hundreds of individual backend views, HAPI utilizes a highly sophisticated metadata-driven architecture.
* [cite_start]The UI is dynamically generated via three core database tables: `map`, `mapdata`, and `maptable`[cite: 91].
* [cite_start]This engine dictates field labels, input types (including WYSIWYG editors and file uploads), and column rendering on the fly, allowing for rapid deployment of new backend modules without touching the core frontend codebase[cite: 94, 95, 96].

### 3. Comprehensive Order-to-Cash (O2C) Module
* [cite_start]**Financial Tracking:** Manages the entire financial lifecycle, linking `orders`, `orderItems`, `invoices`, and `transactions` tables to automatically calculate outstanding balances and client credit[cite: 47, 48, 87, 88].
* [cite_start]**Automated Document Generation:** Utilizes the `mPDF` library to dynamically compile and render branded, professional PDF invoices directly from database variables[cite: 46, 73].
* [cite_start]**Communications:** Integrated with `PHPMailer` for automated outbound SMTP billing communications, and IMAP for ingesting incoming client correspondence[cite: 49, 74, 78].

## Infrastructure & Security Posture
* [cite_start]**Hosting:** Deployed on an on-premise Debian Linux environment hosted via an Oracle Virtual Machine.
* [cite_start]**Database:** MariaDB (relational model utilizing strict `AUTO_INCREMENT` primary keys and foreign key constraints)[cite: 71, 81].
* **Security & Continuity:** Isolated within a private LAN to mitigate external attack vectors. Infrastructure is protected via daily automated block-level VM backups and cron-scheduled database dumps.
* [cite_start]**Stack:** Apache Web Server, PHP 7.4 (hybrid Procedural/OOP architecture), HTML/CSS/JS, jQuery/jQuery UI[cite: 67, 68, 70].
